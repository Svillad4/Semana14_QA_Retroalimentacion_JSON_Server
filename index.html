Zona QA

Antes de entregar, revisa estos puntos. No basta con que el sitio abra: debe funcionar, ser comprensible y tener evidencias.

  1. Prueba datos validos e invalidos en el formulario.
  2. Verifica si JSON Server recibe los registros.
  3. Revisa la consola del navegador.
  4. Evalua si el sitio se entiende en celular.
  5. Documenta bugs con prioridad alta, media o baja.
  6. Aplica correcciones y compara antes/despues.
Atencion: este proyecto tiene detalles intencionales para que puedas practicar diagnostico y mejora.
