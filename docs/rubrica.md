# Rubrica de evaluacion

| Criterio | Porcentaje | Superior | Alto | Basico | Bajo |
|---|---:|---|---|---|---|
| Diagnostico tecnico | 20% | Identifica bugs claros, variados y bien justificados | Identifica varios bugs relevantes | Identifica pocos bugs o con poca explicacion | No evidencia diagnostico real |
| Registro de bugs priorizado | 20% | Documenta minimo 6 bugs con prioridad, pasos, evidencia y estado | Documenta bugs con algunos detalles incompletos | Registra bugs generales sin suficiente evidencia | No presenta registro usable |
| Correcciones aplicadas | 25% | Corrige minimo 4 problemas y demuestra funcionamiento | Corrige varios problemas, aunque falta evidencia completa | Corrige pocos problemas o parcialmente | No aplica correcciones verificables |
| Evidencias antes/despues | 15% | Presenta capturas claras de antes, despues, consola y JSON Server | Presenta evidencias suficientes con algunos vacios | Evidencias incompletas o poco claras | No presenta evidencias |
| Retroalimentacion entre pares | 10% | Realiza coevaluacion tecnica, respetuosa y util | Realiza coevaluacion aceptable | Coevaluacion superficial | No presenta coevaluacion |
| Plan de mejora | 10% | Propone mejoras concretas, priorizadas y viables | Propone mejoras generales | Plan incompleto o poco claro | No presenta plan de mejora |

## Nota para el estudiante

La evaluacion no se basa solamente en que el sitio abra. Tambien cuenta que puedas explicar que probaste, que encontraste, como lo corregiste y por que esa mejora importa.
